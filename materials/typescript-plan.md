# JS + TS 統合資料 章立て設計

## 基本方針

- **最初からTypeScriptで書く**(JSを型なしで学ぶ工程をなくす)
- **ゴール**: TypeScript + React + Tailwind でアプリ開発ができる
- HTML/CSS Day 1〜7 を前提知識とする
- HTML/CSS Day 8〜9 は「JS だけで作りたい人向け」の参考資料として残す(統合教材を進める上では不要)
- Node.js / npm の基本操作は知っている前提(HTML/CSS Day 1で触れる)

---

## 全体構成

```
Phase 0: 導入(2章)
Phase 1: 言語基礎(6章)   ← JS超速習 + TS基本型を統合
Phase 2: 型システム(5章)  ← TS中級
Phase 3: 実践(5章)        ← React + Tailwind + Express
Phase 4: 仕上げ(2章)      ← ベストプラクティス + リソース
```

総章数: 20章

---

## 章立て詳細

### Phase 0 — 導入

| 章 | タイトル | 主な内容 | 元ソース |
|---|---|---|---|
| 第0章 | はじめに | ゴール、資料の使い方、表記ルール | TS ch00 |
| 第1章 | TypeScriptとは / 環境構築 | TSとは何か、プロジェクト作成、tsconfig | TS ch01 + ch02 + ch03 を圧縮 |

**統合の判断**:
- ch00〜ch03 は「TSとは」「環境構築」「tsconfig」で内容が薄めなので2章に圧縮
- JS資料には相当する章がないため TS 側をベースに再構成

### Phase 1 — 言語基礎

| 章 | タイトル | 主な内容 | 元ソース |
|---|---|---|---|
| 第2章 | 変数・型・データ型 | const/let、基本型、型注釈、型推論、any/unknown | JS Day1 + TS ch04 |
| 第3章 | 演算子・制御構文 | 演算子、if/switch、for/while、型変換の罠 | JS Day2 + Day3(前半) |
| 第4章 | 関数 | 関数定義、Arrow Function、型注釈、オーバーロード | JS Day2(後半) + TS ch05(関数部分) |
| 第5章 | 配列・オブジェクト | 配列操作、オブジェクト、分割代入、スプレッド、readonly | JS Day3(後半) + TS ch05(配列・オブジェクト部分) |
| 第6章 | 文字列・スコープ・例外 | 文字列メソッド、スコープ、クロージャ、this、try/catch | JS Day4 |
| 第7章 | クラス・モジュール | クラス、継承、private/readonly、ESモジュール | JS Day5 + TS ch11(クラス・モジュール部分) |

**統合の判断**:
- JS の「型なし版」をそのまま使わず、最初から型注釈を付けて書く
- JS Day1「DevToolsを開く」は HTML/CSS Day1 で既出なので省略
- JS Day2 の「暗黙の型変換」は TS では大半が型エラーになるので、短くまとめる
- `this` はクラスと合わせて第7章に移動
- タプルは型システム寄りなので Phase 2 に移動

---

### Phase 2 — 型システム

| 章 | タイトル | 主な内容 | 元ソース |
|---|---|---|---|
| 第8章 | 非同期処理 | Promise、async/await、型、エラーハンドリング | JS Day6 + TS ch11(非同期部分) |
| 第9章 | 型エイリアス・インターフェース | type/interface、拡張、使い分け | TS ch06 |
| 第10章 | ユニオン型・リテラル型・as const | ユニオン、リテラル、判別可能ユニオン、enum廃止 | TS ch07 |
| 第11章 | ジェネリクス・ユーティリティ型 | ジェネリクス、Partial/Pick/Omit など | TS ch08 + ch09 |
| 第12章 | 型ガードと narrowing | typeof/instanceof、never、ユーザー定義型ガード | TS ch10 |

**統合の判断**:
- 非同期処理は「JSの動作」→「TSの型」を一気に学んだほうが効率的なので Phase 2 冒頭に
- タプルはジェネリクス・ユーティリティ型と一緒に第11章へ(useState の戻り値型の説明に使う)
- JSON・モジュール(JS Day7)はモジュールを第7章に、JSON は第8章の fetch と合わせる

---

### Phase 3 — 実践

| 章 | タイトル | 主な内容 | 元ソース |
|---|---|---|---|
| 第13章 | DOM・Fetch・Vite | Vite環境、DOM型、イベント型、fetch、ToDoアプリ | TS ch12 |
| 第14章 | React + TypeScript 基礎 | コンポーネント、Props、useState、イベント型 | TS ch13(前半) |
| 第15章 | React + TypeScript 応用 | useRef、children、useEffect、ToDoアプリ完成 | TS ch13(後半) |
| 第16章 | Tailwind CSS 入門 | 導入、基本クラス、レスポンシブ、第14〜15章アプリをTailwindで書き直し | 新規 |
| 第17章 | Node.js + Express | APIサーバー、CRUD、フロントと連携 | TS ch14 |

**統合の判断**:
- React 章を2章に分割(1章あたりのボリュームが大きいため)
- Tailwind は React の基礎が固まった直後(第16章)に導入
  → 「素のCSSで作ったToDoをTailwindで書き直す」という流れで差分が体感できる
- JS Day7 の「ToDo CLIアプリ」は第7章(クラス・モジュール)の演習として組み込む
