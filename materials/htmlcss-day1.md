## 1. Webの仕組み — ざっくり理解する(20分)

最初の20分だけ、概念を整理させてください。これを知っているとデバッグが格段に楽になります。

### 1-1. ブラウザ・サーバー・HTTP

普段当たり前に使っている「Webページを見る」という行為は、実は3つの登場人物が動いています。

```
[あなた] ─── 「このページください」 ───→ [サーバー]
[ブラウザ] ←── 「はい、これです」    ───── [Webサイトを置いてる場所]
```

- **ブラウザ**(Chrome、Edge、Safariなど): あなたの手元にあるソフト。Webページを表示する
- **サーバー**: インターネット上のどこかにある、Webサイトのファイルが置かれているコンピューター
- **HTTP**: ブラウザとサーバーが会話するための「言葉のルール」

「`https://example.com` にアクセスする」=「example.com のサーバーに『ページください』とHTTPで頼む」。

### 1-2. Webサイトを構成する3つの言語

ブラウザは、サーバーから受け取った**3種類のファイル**を組み合わせてページを表示します。

| 言語 | 役割 | たとえると |
|---|---|---|
| **HTML** | 構造(何があるか) | 建物の骨組み |
| **CSS** | 見た目(どう見えるか) | 内装・塗装・家具 |
| **JavaScript** | 動き(どう動くか) | 電気・水道・スマート機器 |

このプランでは:
- **Day 1〜3**: HTML(骨組み)
- **Day 4〜7**: CSS(見た目)
- **Day 8〜9**(参考): JavaScript と組み合わせる ※ 通常は TypeScript 本編へ進むので必須ではありません

を順番にやっていきます。動き(JavaScript / TypeScript)は、Day 7 の修了後に **TypeScript 本編** で本格的に学びます。

### 1-3. ファイルの拡張子

| 拡張子 | 言語 |
|---|---|
| `.html` | HTML |
| `.css` | CSS |
| `.js` | JavaScript |
| `.svg` / `.png` / `.jpg` | 画像 |

これからの章では、これらを VS Code で書いていきます。

## 3. HTML の基本構造(20分)

先ほど書いた「おまじない」を、1行ずつ理解していきましょう。

### 3-1. 全体の骨組み

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <!-- ページの情報(タイトル、文字コードなど) -->
</head>
<body>
  <!-- 実際に画面に表示される内容 -->
</body>
</html>
```

それぞれの役割:

- **`<!DOCTYPE html>`**: 「これは HTML ですよ」という宣言。最初に必ず書く
- **`<html lang="ja">`**: HTMLのルート要素。`lang="ja"` で「日本語のページ」と教える
- **`<head>`**: ページの**情報**(画面には表示されない)
- **`<body>`**: 画面に**表示される内容**

### 3-2. タグの書き方

HTML は **タグ** で要素を表します。

```html
<h1>これは見出し</h1>
```

- `<h1>` = **開始タグ**
- `これは見出し` = **中身**(コンテンツ)
- `</h1>` = **終了タグ**(`/` がつく)

`<h1>` で囲まれた部分が「**h1 という意味の要素**」になります。

### 3-3. 自己完結タグ(空要素)

中身が無いタグもあります。終了タグ無しで、1つで完結します。

```html
<img src="photo.jpg" alt="写真">
<br>
<hr>
```

- `<img>`: 画像
- `<br>`: 改行
- `<hr>`: 水平線

(古い書き方として `<img />` や `<br />` のように `/` を付けることもありますが、現代のHTML(HTML Living Standard)ではどちらでもOKです)

### 3-4. 属性

タグには**属性**で追加情報を持たせられます。

```html
<a href="https://example.com">リンク</a>
<img src="photo.jpg" alt="花の写真">
<html lang="ja">
```

- `href="..."` リンク先のURL
- `src="..."` 画像ファイルのパス
- `alt="..."` 画像が表示できないときの代替テキスト
- `lang="..."` 言語

形式は **`属性名="値"`**。属性は**スペースで区切って複数**書けます。

### 3-5. head の中身を詳しく

```html
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>はじめての HTML</title>
</head>
```

- **`<meta charset="UTF-8">`**: 文字コードの指定。日本語を正しく表示するために必須
- **`<meta name="viewport" content="...">`**: スマホで正しく表示するためのおまじない(レスポンシブの章で詳しく)
- **`<title>`**: ブラウザのタブに表示される名前

これら3つは**HTMLの最初に必ず書く**3点セットとして覚えてください。

## 4. 主要なタグを覚える(40分)

ここからが本番。よく使うタグを順番に紹介します。

### 4-1. 見出し: h1〜h6

```html
<h1>大見出し(ページのタイトル)</h1>
<h2>中見出し(セクションのタイトル)</h2>
<h3>小見出し</h3>
<h4>小小見出し</h4>
<h5>めったに使わない</h5>
<h6>もっと使わない</h6>
```

**重要なルール**:
- 数字が小さいほど大きく表示される
- **`<h1>` はページに1つだけ**が原則
- **階層を飛ばさない**(`<h1>` の次は `<h2>`、いきなり `<h3>` はNG)

これは**SEO**(検索エンジンに見つけてもらいやすくする)や**アクセシビリティ**(視覚障害者向けの読み上げソフトでの理解しやすさ)にも関わります。

### 4-2. 段落: p

「**p**aragraph(段落)」の略。本文の文章を入れる場所。

```html
<p>これは1つ目の段落です。</p>
<p>これは2つ目の段落です。改行は自動的に入ります。</p>
```

**改行を入れたいだけ**で `<br>` を多用するのは避け、段落ごとに `<p>` で分けるのが原則。

### 4-3. リンク: a

「**a**nchor(アンカー、錨)」の略。クリックで別のページに飛びます。

```html
<a href="https://example.com">example.com に行く</a>
<a href="about.html">同じサイト内のページへ</a>
<a href="https://example.com" target="_blank">新しいタブで開く</a>
```

- `href="..."`: リンク先
- `target="_blank"`: 新しいタブで開く

### 4-4. 画像: img

```html
<img src="photo.jpg" alt="桜の写真">
```

- `src="..."`: 画像ファイルのパス
- **`alt="..."`は必須**: 画像の説明文。視覚障害者の読み上げや、画像が表示できないときに使われる

```html
<!-- 装飾用の画像なら空でOK -->
<img src="decoration.svg" alt="">
```

「**装飾画像のときは `alt=""`(空)**」と覚えてください。`alt`属性自体は省略しないこと。

### 4-5. リスト: ul / ol / li

「**u**nordered **l**ist(順不同リスト)」と「**o**rdered **l**ist(順序付きリスト)」、中の項目は「**l**ist **i**tem」。

```html
<!-- 順不同リスト(箇条書き、・が付く) -->
<ul>
  <li>りんご</li>
  <li>バナナ</li>
  <li>みかん</li>
</ul>

<!-- 順序付きリスト(番号が付く) -->
<ol>
  <li>朝起きる</li>
  <li>顔を洗う</li>
  <li>朝ごはんを食べる</li>
</ol>
```

`<li>` は必ず `<ul>` か `<ol>` の中に入れます。

### 4-6. 強調: strong / em

文章の一部を強調したい時。

```html
<p>これは <strong>とても重要</strong> な情報です。</p>
<p>彼は <em>本当に</em> 急いでいた。</p>
```

- **`<strong>`**: 重要性を示す(太字で表示されることが多い)
- **`<em>`**: 強調(斜体で表示されることが多い)

> ⚠️ **昔の書き方は使わない**
>
> 古いWebでは `<b>`(太字)や `<i>`(斜体)が使われていましたが、これらは「**見た目**」を表すタグでした。現代では「**意味**」を表す `<strong>` / `<em>` を使います。見た目だけ変えたい場合は CSS で。

### 4-7. 改行と水平線: br / hr

```html
<p>これは1行目<br>これは2行目</p>

<hr>

<p>水平線の下にある段落</p>
```

- `<br>`: 改行(段落の中で行を変えたいとき)
- `<hr>`: 水平線(セクションの区切りなど)

両方とも**自己完結タグ**(終了タグ不要)。

### 4-8. コメント

メモを残せます。ブラウザには表示されません。

```html
<!-- これはコメントです。表示されません -->
<p>これは表示されます</p>
```

`<!--` から `-->` までがコメント。

## 5. ブラウザの DevTools を開く(15分)

これからの学習で**最も大事なツール**を紹介します。

### 5-1. DevTools とは

ブラウザに組み込まれた**開発者ツール**。HTML/CSS/JavaScript の中身を見たり、変更したりできます。

**Chrome / Edge / Firefox** のいずれでも:

| ショートカット | アクション |
|---|---|
| `F12` | DevTools を開く |
| `Ctrl + Shift + I`(Mac: `Cmd + Option + I`) | DevTools を開く |
| 要素を右クリック → 「検証」 | その要素にカーソルを当てた状態で開く |

### 5-2. Elements パネル(最重要)

DevTools の **「Elements」タブ**(または「要素」タブ)を開いてください。

ここには:
- 現在のページの**HTML構造**が左側に
- 選択した要素の**CSS**が右側に

表示されます。

### 5-3. やってみる

今作った `index.html` を開いて、F12 で DevTools を開いてください。

1. Elements パネルで `<h1>` タグをクリック
2. 右側に CSSが表示される(まだ何もスタイル付けていないので、デフォルトのもの)
3. `<h1>` をダブルクリックして、テキストを「**こんにちは、世界!**」に変えてみる

**ブラウザの表示が即座に変わる**はずです。

> **注意**: DevTools での変更は**ブラウザの一時的な表示だけ**を変えます。実際のファイルは変更されません。リロードすると元に戻ります。

### 5-4. これからの使い方

これから学習中、何度も DevTools を使います:

- **「なぜ思った通りに表示されないのか?」を調べる**
- **CSS を試行錯誤する**(右側のスタイル欄で値を変えると即反映)
- **要素の位置やサイズを確認する**

「**コードが思った通りに動かないとき、まず DevTools を開く**」が現代Web開発の鉄則です。

> 🎯 **コラム: なぜ Live Server を使うのか**
>
> 「ファイルをダブルクリックで開けばいいのでは?」と思うかもしれません。
>
> ローカルのHTMLファイルをダブルクリックで開くと、URLが `file:///Users/.../index.html` のような形になります。これでも見られはしますが、後の章で:
>
> - JavaScript の一部機能が動かない(セキュリティ制限)
> - 画像やCSSのパスがうまく解決できない
> - リロードを毎回手動で押す必要がある
>
> といった不便があります。
>
> Live Server は **ローカルに小さな Web サーバーを立てて、`http://127.0.0.1:5500` のような URL で配信**してくれます。「本物のWebサーバーで動かしている」状態になるので、後々のトラブルがずっと減ります。
