- HTML/CSS Day 6（CSS Grid・レスポンシブデザイン）を学習・演習完了
  - 学び: Grid は2次元（行×列）レイアウト。Flexboxは1次元
  - 学び: `fr` 単位で余ったスペースを比率で分ける。`repeat(3, 1fr)` で3列均等
  - 学び: `repeat(auto-fit, minmax(250px, 1fr))` がカードグリッドの現代定石。メディアクエリ不要で列数が自動変化
  - 学び: `grid-template-areas` でレイアウトを視覚的に定義。`grid-area: 名前` で各要素を配置
  - 学び: `grid-area` の名前はクラス名ではなく任意の名前。セマンティックタグ（`header`/`main`など）は1個しか出ないのでクラスなしで直接指定できる
  - 学び: Flexbox vs Grid の使い分け → 1次元ならFlex、2次元ならGrid、入れ子で組み合わせるのが現代の主流
  - 学び: モバイルファーストはスマホ用を先に書き、`@media (min-width: ...)` でPC用を上書き
  - 学び: `clamp(最小, 推奨, 最大)` でフォントサイズが画面幅に応じてスムーズに変化。メディアクエリ3つが1行になる
  - 演習: `sandbox/day6/index.html` + `style.css` でGrid レイアウト・レスポンシブ・clamp を実装
- HTML/CSS Day 5（Flexbox）を学習・演習完了
  - 学び: `display: flex` は**親要素**に書く（子に書いても効かない）
  - 学び: `gap` で要素間の余白を一括設定。昔の `margin-right` より現代は `gap` 一択
  - 学び: `flex-direction: column` で縦並び、`row-reverse` で逆順横並び
  - 学び: `justify-content` = 主軸（横）方向の整列、`align-items` = 交差軸（縦）方向の整列
  - 学び: 縦横センタリングは `justify-content: center` + `align-items: center` の2行で解決（CSS界の長年の難題がこれだけで終わる）
  - 学び: `justify-content: space-between` でナビバーの「ロゴ左・メニュー右」レイアウトが作れる
  - 学び: `flex-wrap: wrap` + `width` or `flex: 1 1 250px` で画面幅に応じて列数が自動変化するカードグリッド
  - 学び: `flex: 1` で余ったスペースを埋める、`flex-shrink: 0` で縮ませない（アバター・ロゴなどに使う）
  - 学び: Flexboxは入れ子にできる（アバター内の文字センタリングにもFlexboxを使う）
  - 詰まった: カードの横並びで `justify-content: space-between` を使ったが、`.info` に `flex: 1` を付ける方が意味として正確
  - 演習: `sandbox/day5/index.html` + `style.css` でプロフィールカードを自力で作成
- HTML/CSS Day 3（フォームと入力要素）を学習・演習完了
  - 学び: `<form>` でフォーム全体を囲み、`<label for="id">` で入力欄に必ずラベルを付ける
  - 学び: `<input type="...">` の種類で見た目と機能が変わる（email/password/number/tel/radio/checkbox など）
  - 学び: `required` / `minlength` / `min` / `max` でJavaScriptなしのネイティブバリデーション
  - 学び: `min` はnumber用、テキストの文字数制限は `minlength`（ここで詰まった）
  - 学び: ラジオボタンは `name` を揃えることでグループ化、`value` を必ず書く
  - 学び: 重複idはNG。ラジオ/チェックボックスを `<label>` で囲む書き方なら `id` は不要
  - 学び: `<fieldset>` + `<legend>` で関連する入力欄をグループ化（読み上げソフトにも優しい）
  - 学び: `<dialog>` 要素でJSなしに近いモーダルが作れる。`<form method="dialog">` でボタンを押すと閉じる
  - 学び: `event.preventDefault()` するとネイティブバリデーションも止まる。validationとdialogの両立はDay 8のJS統合で解決
  - 演習1: `sandbox/day3/index.html` に会員登録フォームを作成
  - 演習2: `sandbox/day3/contact.html` にお問い合わせフォーム＋確認ダイアログを作成
- HTML/CSS Day 4（CSS基礎・セレクタ・ボックスモデル）を学習・演習完了
  - 学び: CSSはHTMLに適用する方法が3つある（インライン・styleタグ・外部ファイル）。外部ファイルが基本
  - 学び: セレクタの種類——要素・クラス（`.cls`）・ID（`#id`）・属性・疑似クラス（`:hover` `:focus-visible`）
  - 学び: CSSはIDセレクタを使わないのが現代のベストプラクティス（詳細度が極端に上がりトラブルの元）
  - 学び: カスケード＝後に書いたルールが勝つ（詳細度が同じなら）
  - 学び: 詳細度の順はインライン > ID > クラス > 要素。`!important` は最終手段
  - 学び: 継承——文字系プロパティ（color/font-familyなど）は子に引き継がれる。見た目系（background/marginなど）は引き継がれない
  - 学び: ボックスモデル——各要素はcontent・padding・border・marginの4層で構成される
  - 学び: `box-sizing: border-box` を全要素に適用するのが現代の標準。CSSファイルの先頭に必ず書く
  - 学び: CSS変数——`:root` に `--変数名: 値` で定義し、`var(--変数名)` で使う。デザイントークン管理の基本
  - 学び: 色指定はhex（`#3b82f6`）が実務の基本。`rgba()` は透明度を使うとき、`oklch()` は次世代の書き方
  - 詰まった: `max-height` と `max-width` を間違えた。幅の上限は `max-width`
  - 詰まった: CSS変数を定義したのに直接値を書いてしまった（変数を使う意味がなくなる）
  - 演習: `sandbox/day4/blog.html` + `style.css` でスタイル付きブログページを作成

  - 学び: 基本は `const`、再代入が必要なときだけ `let`、`var` は使わない
  - 学び: 型推論があるのでローカル変数は型注釈不要。「境界に型を書く」→ 関数の引数は必須、ローカル変数は推論に任せる
  - 学び: `===` を使う（`==` は型変換が入るので予期しないバグの原因になる）

- TypeScript ch01（TypeScriptとは・環境構築・tsconfig）完了
  - 学び: TypeScript = JavaScript + 型システム。最終的にJSに変換されて実行される
  - 学び: `npm start`/`npm run dev` は型チェックしない。型エラーはVS Codeの赤線か `npm run typecheck` で確認
  - 学び: `strict: true` は8つの厳格チェックの一括スイッチ。`noImplicitAny`（暗黙のany禁止）と `strictNullChecks`（null/undefinedを別の型として扱う）が特に重要
  - 学び: `noUncheckedIndexedAccess: true` で配列インデックスアクセスの型が `string | undefined` になり、うっかりアクセスを防げる
  - 学び: `noEmit: true` で tsc は型チェックだけ担当し、実行は tsx が担当する役割分担
  - 実験: `strict: false` にすると型違反コードに赤線が出なくなる → TypeScriptを使う意義の大半が `strict: true` にある
  - 環境: `sandbox/typescript-learning/01-basics/` を構築。`npm start` で `Hello, TypeScript!` 動作確認
- HTML/CSS Day 7（現代CSS応用・アニメーション）を学習・演習完了 → HTML/CSS編（Day 1〜7）完走
  - 学び: CSS Nesting で関連スタイルを1ブロックにまとめられる。子要素は `&` 不要、疑似クラス・複数クラスは `&` が必要
  - 学び: `:has()` で「子要素の状態に応じた親のスタイル変更」がJS不要で書ける（チェックボックス取り消し線・フォームエラー強調など）
  - 学び: Container Queries はコンポーネント自身の幅でレスポンシブを制御する仕組み（概念のみ、React編で本格活用）
  - 学び: `transition: プロパティ 時間` で変化を滑らかに。複数プロパティはカンマ区切り。`transition: all` はパフォーマンス上非推奨
  - 学び: `transform: translateY(-4px)` でホバー浮き上がり。`width/height` より `transform` が滑らかで高速
  - 学び: `@keyframes` で動きをステップ定義し `animation: 名前 時間 イージング` で適用。スピナー・スライドインは定番パターン
  - 学び: `@media (prefers-reduced-motion: reduce)` に `*` でグローバル対応するとアクセシビリティ配慮が1ブロックで完結
  - 学び: CSS変数 + `@media (prefers-color-scheme: dark)` でダークモード対応。色は必ず変数経由で書く
  - 学び: `color-mix(in oklch, var(--primary), black 15%)` で派生色を自動生成。主色を変えるだけで全派生色が連動する
  - 詰まった: `transition: background 0.2s transform 0.2s` → カンマ区切りを忘れると全部無効になる
  - 詰まった: `translate(-2px)` は横移動、上に浮かせるなら `translateY(-2px)`
  - 詰まった: `rgba()` の第4引数は 0〜1 の小数（`15` ではなく `0.15`）
  - 演習: `sandbox/day7/index.html` + `style.css` で Nesting・:has()・transition・animation を実装

## 😅 詰まったこと・失敗

> ※Zenn記事のネタになる宝の山。正直に書く。

- CSS: `transition` は変化するプロパティをカンマ区切りで列挙する必要がある（省略すると全部無効）
- CSS: `translate(-2px)` は横移動。上に浮かせるには `translateY(-2px)`
- CSS: `rgba()` の第4引数は 0〜1 の小数（`15` と書いてしまい透明度が狂った）
- HTML: `min` は number 用、テキストの文字数制限は `minlength`（混同した）
- TS: `let` で初期値なし宣言に型注釈を書かないと暗黙の `any` になる
- TS: 宣言した変数と違う変数を渡してしまった（`userId2` を作ったのに `userId` を使っていた）
