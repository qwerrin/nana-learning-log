## 😅 詰まったこと・失敗

> ※ここはZenn記事のネタになる宝の山。正直に書く。

- `Cannot redeclare block-scoped variable 'sum'` エラー
  - 別ファイル（`03-control.ts` と `04-functions.ts`）で同じ変数名を使っていてぶつかった
  - 原因: `import`/`export` のない TS ファイルは全部グローバルスコープ扱いになる
  - 解決: ファイル末尾に `export {};` を追加してモジュール宣言する
- コールバック関数の「向き」を取り違えていた
  - 「コールバック関数とは?」と聞かれて「他の関数を引数として受け取る関数」と答えた
  - 正しくは: 受け取る側 = 高階関数、渡される側 = コールバック関数
  - 「コール(呼び出し)をバック(あとで)する」= 受け取った側があとで呼ぶための関数、と覚えた
- pipe で「前の結果」ではなく「最初の値」を渡し続けてしまった
  - `for` で `total = f(x)` と書いてしまい、毎回最初の値で処理 → 結果が 9 になる（期待: 49）
  - 原因: 累積処理では「前回の結果」を次に渡す必要があると気づいていなかった
  - 解決: `total = f(total)` — ループ内では毎回更新される変数を渡す
- `reduce` の構造が最初理解できなかった
  - `(acc, current) => ...` の意味と第2引数の初期値の役割が掴めなかった
  - `for` 版との対応表で整理して理解（`acc` = `let total`、第2引数 = 初期値、`(acc, f) => f(acc)` = `total = f(total)`）
  - 慣れないうちは `for` で書いて、書けたら `reduce` に直す、で十分
- デフォルト引数 vs オプショナル引数の型の違いを取りこぼした
  - 「省略時の動作が違う」は言えたが、`greeting` の型が `string` か `string | undefined` かの視点が欠けた
  - デフォルト引数 `(x = "default")` → 型は `string`
  - オプショナル引数 `(x?: string)` → 型は `string | undefined`
- `map` + 三項演算子 + スプレッドのパターンが最初読めなかった
  - `t.priority === "high" ? { ...t, done: true } : t` の3つが組み合わさっていて、何をしているか一度に把握できなかった
  - 分解して理解: ① `map` で全要素を処理 → ② 三項演算子で「変える/変えない」を分岐 → ③ スプレッドで「一部だけ変えたコピー」を作る
  - React の状態更新でほぼ毎回使うパターンなので、体に染み込ませる必要がある
