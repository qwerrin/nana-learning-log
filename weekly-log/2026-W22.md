### 火曜
- TypeScript ch04（関数）を学習 → `sandbox/typescript-learning/01-basics/src/04-functions.ts`
  - 行動: Arrow Function・型注釈・デフォルト/オプショナル/残余引数・関数型・コールバック・パイプライン関数を実装
  - 学び: Arrow Function が現代 TypeScript の標準。`const f = (x: T): R => ...` の形
  - 学び: 引数には必ず型を書く（推論できないため）。戻り値は省略可
  - 学び: デフォルト引数 `(x = "default")` は `string` 型。オプショナル引数 `(x?: string)` は `string | undefined` 型
  - 学び: 関数型 `(x: T) => R` でコールバックを引数に渡せる。`map`/`filter`/`reduce` と同じ仕組み
  - 学び: `export {}` を書かないとファイルがグローバルスコープ扱いになり、別ファイルの変数名と衝突する
  - 詰まった: `Cannot redeclare block-scoped variable 'sum'` → `import`/`export` のない TS ファイルは全部同じスコープになる。`export {}` を末尾に追加して解決
- ch04 理解確認（/study）: 3問中2問○・1問△
  - △ だったのは「デフォルト引数 vs オプショナル引数の型の違い」（動作の違いは言えたが、型が `string` か `string | undefined` かの視点が欠けた）

### 木曜
- 関数の型注釈を JS/TS 対比で復習（引数2つ・引数なし・戻り値なし）
  - 学び: 引数の型 → `引数名: 型`、戻り値の型 → `)` の後ろに `: 型`、何も返さないなら `: void`
  - 学び: 引数の型だけは省略不可（推論できないため）。戻り値は推論されるが複雑な関数では明示が安全
